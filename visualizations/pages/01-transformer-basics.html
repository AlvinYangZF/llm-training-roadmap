Architecture Decoder-Only Transformer

The modern LLM architecture uses a decoder-only transformer. Data flows bottom to top, through repeated transformer blocks.

Input Tokens → Token Embedding + Positional Encoding
↓
× N Transformer Blocks
↓
Layer Normalization
↓
Multi-Head Self-Attention (Q, K, V)
↓
Residual Add + Layer Normalization
↓
Feed-Forward Network (FFN)
↓
Residual Add + Layer Normalization
↓
End of Block — Repeat × N
↓
Linear Projection → Softmax → Next Token Probabilities

Concepts Key Components

Each transformer block contains these core building blocks working together.

Embedding Layer

Converts discrete token IDs into dense vector representations. A learned lookup table maps each token in the vocabulary to a d_model-dimensional vector. Positional encodings are added to inject sequence order information, since the architecture has no inherent notion of position.

Multi-Head Self-Attention

The core mechanism that lets each token attend to every other token in the sequence. The input is projected into Query, Key, and Value matrices. Attention scores are computed as scaled dot products of Q and K, then used to weight V. Multiple heads capture different relationship patterns in parallel.

Feed-Forward Network

A position-wise two-layer MLP applied independently to each token. Typically expands the dimensionality by 4x (e.g., d_model → 4 × d_model → d_model) with a nonlinear activation (ReLU or GELU) in between. This provides the model's capacity for complex transformations.

Residual Connections & Layer Norm

Skip connections add the input of each sub-layer to its output (x + SubLayer(x)), enabling gradient flow through deep networks. Layer normalization stabilizes training by normalizing activations across the feature dimension. Modern LLMs typically use pre-norm (normalize before the sub-layer).

Concept Multi-Head Attention

Instead of performing a single attention function, the model runs h parallel attention heads. Each head learns different aspects of the relationships between tokens — syntax, semantics, positional patterns, etc.

MultiHead(Q, K, V) = Concat(head1, ..., headh) · WO
Input X (d_model = 512)
↓ Linear projections split into h=8 heads
↓ Concatenate all heads
Concat(head_1, ..., head_8) — dim = 512
↓ Final linear projection WO
Multi-Head Output (d_model = 512)

Each head operates on d_k = d_model / h = 64 dimensions. The parallel heads are computationally equivalent to a single full-dimension attention but capture richer patterns.
