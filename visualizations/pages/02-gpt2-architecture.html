1 Encoder-Decoder vs Decoder-Only

The original Transformer (Vaswani et al., 2017) uses both an encoder and a decoder with cross-attention connecting them. GPT-2 simplifies this by keeping only the decoder stack, removing the encoder entirely and the cross-attention layers that connected them.

Original Transformer Vaswani et al., 2017

Input Embedding
↓
Encoder Self-Attention
↓
Encoder FFN
↓
x N Encoder Layers
⇓
Output Embedding
↓
Masked Self-Attention
↓
Cross-Attention
↓
Decoder FFN
↓
Linear + Softmax

GPT-2 Radford et al., 2019

Encoder Stack REMOVED
↓
Token + Position Embedding
↓
Masked Self-Attention KEPT
↓
Cross-Attention REMOVED
↓
Feed-Forward Network KEPT
↓
Output Projection

GPT-2 processes input and output in a single pass through the decoder stack. No encoder needed for language modeling.

2 Pre-LayerNorm vs Post-LayerNorm

The original Transformer applies LayerNorm after the residual addition (Post-LN). GPT-2 moves LayerNorm before each sublayer (Pre-LN), which produces more stable gradients and enables training of much deeper networks.

Post-LayerNorm Original

Input (x)
↓
Sublayer(x)
↓
Add (x + Sublayer(x))
↓
LayerNorm
↓
Output

Pre-LayerNorm GPT-2

Input (x)
↓
LayerNorm(x)
↓
Sublayer(LN(x))
↓
Add (x + Sublayer(...))
↓
Output
Post-LN issues:
Gradient magnitudes vary wildly across layers, making deep networks hard to train without careful warmup.
Pre-LN benefits:
Gradients stay well-behaved through many layers. Enables stable training of 48+ layer models without warmup.

4 Weight Tying Visualization

GPT-2 shares the same weight matrix between the input token embedding and the output projection layer. The output projection is simply the transpose of the embedding matrix.

We (Embedding)
vocab_size × d_model
50,257 × 768
↔
Wo = WeT
Same weights!
Wo (Output Projection)
d_model × vocab_size
768 × 50,257
~30% fewer parameters Eliminates a separate 50,257 × 768 output matrix (~38.5M parameters saved in GPT-2 Small).
Acts as regularization Forces the model to learn embeddings that work well for both understanding input and predicting output.
Better generalization Shared representations create a more coherent semantic space where input and output meanings align.
Faster convergence Fewer parameters to learn means the model can converge more quickly during training.
