Core Memory Hierarchy

The key insight behind FlashAttention: GPU memory has a hierarchy. SRAM is fast but tiny; HBM is large but slow. Standard attention bounces data through HBM for every intermediate result, creating a massive IO bottleneck.

SRAM (On-Chip)
~20 MB  |  ~19 TB/s bandwidth
↓↑   ~13x bottleneck   ↓↑
HBM (Off-Chip)
40-80 GB  |  ~1.5 TB/s bandwidth

FlashAttention keeps computation in SRAM tiles, avoiding redundant HBM round-trips.

Compare Standard vs FlashAttention

Standard Attention

  • Compute S = Q KT → write to HBM
  • Read S from HBM → compute softmax P → write to HBM
  • Read P from HBM → compute O = P V → write to HBM
  • Every N x N intermediate materialized in HBM
IO = O(N2)

FlashAttention

  • Load Q, K, V tiles into SRAM
  • Compute attention in SRAM — no intermediate writes
  • Use online softmax to accumulate across tiles
  • Write only final O blocks back to HBM
IO = O(N2d / M)

Where N = sequence length, d = head dimension, M = SRAM size. FlashAttention reduces IO by a factor of M/d.

Key Trick Online Softmax

Standard softmax requires seeing all scores before normalizing. FlashAttention uses online softmax to update the running result as each K-block tile is processed:

1. Update running max:
m_new = max(m_old, max(S_tile))
2. Update normalizer:
l_new = exp(m_old - m_new) × l_old + sum(exp(S_tile - m_new))
3. Update output:
O = (l_old/l_new) × exp(m_old - m_new) × O_old + (1/l_new) × exp(S_tile - m_new) × V_tile
This produces the exact same result as standard softmax — no approximation. The trick is rescaling previous partial sums when the running max changes.
