Problem The Interference Problem

When prefill (compute-bound) and decode (memory-bound) share the same GPU, they interfere with each other. Long prefill jobs block decode responses, causing unpredictable latency spikes.

Prefill (compute-heavy)
Decode (memory-heavy)
Waiting / blocked

Coupled: Prefill blocks Decode → unpredictable latency

Solution Disaggregated Architecture

Separate GPU pools are dedicated to each phase. Prefill nodes crunch prompts in parallel; decode nodes generate tokens at steady throughput. KV cache is transferred between pools.

Prefill GPU Pool
Decode GPU Pool
KV Cache Transfer

Disaggregated: Each pool optimized for its workload

Efficiency Resource Utilization Comparison

In coupled mode, compute and memory bandwidth oscillate wildly. With disaggregation, each pool maintains near-peak utilization of its bottleneck resource.
