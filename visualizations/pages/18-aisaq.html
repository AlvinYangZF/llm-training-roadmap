Memory

The DiskANN Memory Problem

DiskANN stores all PQ-compressed vectors in DRAM for distance computation during beam search. For SIFT1B (1 billion vectors), this requires 31 GB of RAM. AiSAQ eliminates this entirely, using only ~10 MB regardless of dataset size.

Architecture

Node Chunk Architecture

The key insight: AiSAQ embeds neighbor PQ vectors directly inside each node chunk on SSD. When a node is read from storage during beam search, all needed PQ vectors arrive with it — no DRAM cache required.

RAG

Index Switching for RAG

Multi-source RAG systems frequently switch knowledge bases. DiskANN must reload all PQ vectors into DRAM (119 ms). AiSAQ switches in 0.3 ms with shared PQ centroids — a 400x improvement.
