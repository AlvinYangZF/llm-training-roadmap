# H2O Heavy-Hitter Oracle Visualization Design

## Reference
- Paper: "H2O: Heavy-Hitter Oracle for Efficient Generative Inference of Large Language Models", Zhang et al., NeurIPS 2023
- Local: research/papers/12_H2O_深度解读.md

## Key Concepts to Visualize
1. 95%+ attention sparsity — vast majority of attention weights are near zero at any given step
2. Power-law distribution — a small fraction of tokens accumulate most of the attention mass
3. Heavy-hitter tokens — the ~5% of tokens that receive disproportionate attention across layers
4. Hybrid eviction — combining attention sinks (initial tokens) with heavy-hitter retention for KV eviction

## Visualization Sections

### Section 1: Attention Sparsity Heatmap
- **Type**: Canvas heatmap
- **Shows**: N*N attention matrix where most cells are dark (near-zero weight) and a sparse set of cells are bright (high weight), demonstrating the 95%+ sparsity pattern across a real layer
- **Interaction**: Hover cells to see exact attention weight values; slider to switch between layers (1-32) showing sparsity varies by depth; threshold slider to highlight only weights above a cutoff
- **Data**: Example from OPT-6.7B: layer 8 attention matrix for 128-token sequence; ~96% of weights below 0.01; top 5% of tokens capture ~80% of attention mass

### Section 2: Power-Law Distribution Bar Chart
- **Type**: Sorted bar chart
- **Shows**: Tokens sorted by cumulative attention score (descending), revealing the power-law curve where a few tokens dominate; vertical dashed line marks the "heavy-hitter" threshold
- **Interaction**: Drag the threshold line to adjust the heavy-hitter cutoff; displays percentage of total attention captured by tokens left of the threshold
- **Data**: Top 5% tokens: ~80% attention mass; top 10%: ~90%; top 20%: ~97%; long tail of near-zero tokens

### Section 3: KV Eviction Animation
- **Type**: Canvas animation
- **Shows**: Sequence of KV cache slots as colored blocks; as new tokens arrive and cache is full, the eviction policy removes the lowest-scored non-sink token; heavy-hitters persist, low-attention tokens get evicted
- **Interaction**: Auto-play token-by-token generation; evicted tokens flash red before disappearing; retained heavy-hitters glow green; step controls for forward/back
- **Data**: Cache budget = 20% of sequence length; eviction criteria: cumulative attention score; sink tokens (positions 0-3) always retained

### Section 4: Interactive Budget Slider
- **Type**: Interactive slider with live metrics
- **Shows**: Slider controlling KV cache budget (5%-100% of full cache); as budget decreases, perplexity curve and throughput bar update in real time, showing the quality-efficiency tradeoff
- **Interaction**: Drag slider to set budget; perplexity chart and throughput multiplier update live; green zone indicates <1% quality loss, yellow for 1-5%, red for >5%
- **Data**: OPT-6.7B on WikiText-2: 100% budget PPL=10.8, 20% budget PPL=11.1 (<3% degradation), 10% budget PPL=12.5, 5% budget PPL=15.2; throughput scales ~inversely with budget

### Section 5: Sink + Heavy-Hitter Pattern
- **Type**: Comparison grid
- **Shows**: Three retention strategies side by side: (1) recent-window only (loses long-range), (2) heavy-hitter only (loses initial context), (3) H2O sink+HH (keeps both initial sinks and dynamic heavy-hitters); each shows which token positions are retained in a sequence bar
- **Interaction**: Click each strategy to see its perplexity result; animate a long sequence scrolling through each strategy to show what gets dropped
- **Data**: Window-only PPL: 14.2, HH-only PPL: 12.8, H2O (sink+HH) PPL: 11.1 at 20% budget; sink tokens = first 4 positions

## Technical Notes
- Attention heatmap uses Canvas with custom colormap (viridis-like) for smooth gradient rendering
- Power-law chart uses HTML div bars for accessibility (screen reader compatible)
- Eviction animation uses Canvas with sprite-based token blocks for smooth removal transitions
- Mobile: heatmap supports pinch-zoom; budget slider is full-width; comparison grid stacks vertically
